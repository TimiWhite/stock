# A-Share Shipan Line Picker

## Overview

执行“先筛选、再确认、后执行”的短线流程。  
先按试盘线定义筛掉不合规标的，再结合生命线突破确认买点，最后用固定止损和见顶信号管理退出。

完整规则和检查清单见 [references/shipan-line-playbook.md](references/shipan-line-playbook.md)。

## Input Requirements

要求至少提供以下数据后再开始筛选：
- 标的列表（代码、名称）
- 最近 90 个交易日的日线 OHLCV（开高低收量）
- 交易所板块信息（主板、创业板、科创板）

缺少数据时，先明确缺口并请求最小补充字段，不在数据不足时给出确定性买卖建议。

## Workflow

1. 识别试盘线日  
只保留同时满足以下条件的标的：
- 当日成交量 >= 前一日 3 倍
- 位置处于相对低位，且前期有 1-3 个月震荡整理
- 当日为真阳线，优先保留突破整理区上沿的个股

2. 绘制生命线  
将试盘线当日收盘价作为生命线，后续以生命线突破作为执行锚点。

3. 确认买点  
从试盘线次日开始，观察是否出现 K 线实体向上突破生命线。  
优先在尾盘（约 14:45 后）确认有效突破，规避盘中假突破。

4. 设置止损  
- 主板：生命线下方 3%-5%
- 创业板/科创板：生命线下方 5%-8%

5. 跟踪失效与持有  
突破后 5 个交易日内未出现大阳线或涨停，标记为“拉升意愿不足”，降级为观察标的，不盲目加仓追涨。

6. 执行止盈  
出现以下见顶信号时，分批或纪律性离场：
- 放巨量阴线
- 放巨量十字星或长上影线
- 连续 2-3 天长上影线
- 巨量达到试盘线量能约 3 倍或以上

## Output Template

每次分析按以下结构输出：

1. `市场环境`
- 大盘/板块是否支持短线突破策略

2. `筛选结果`
- 符合试盘线条件的标的列表
- 每只标的的试盘线日期、试盘线收盘价（生命线）、突破状态

3. `交易计划`
- 入场条件（是否需尾盘确认）
- 止损位（价格与百分比）
- 第一/第二止盈触发条件

4. `风险提示`
- 不满足条件的关键原因
- 计划失效条件与纪律要求

## Guardrails

- 不承诺收益，不给出“必涨”表述
- 不在缺失关键数据时输出确定性结论
- 每次建议必须包含止损位和失效条件
- 优先保证盈亏比和回撤控制，再考虑收益空间
